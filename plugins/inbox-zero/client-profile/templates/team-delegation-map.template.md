# Team Delegation Map

> Populated during onboarding. The agent uses this to route emails to the right team member.
> Update this file as team members or responsibilities change.

## How Delegation Works

When an email arrives, the agent checks:
1. Is a team member CC'd? -> Route to them
2. Does the subject/content match a team member's domain keywords? -> Route to them
3. Is a team member already replying in the thread? -> It's their thread
4. Did the exec forward it with no extra instructions? -> EA handles

## Default Routing

| Domain | Routes To | Keywords |
|--------|-----------|----------|
| Finance | Exec review required | invoice, billing, payment, expenses, budget |
| HR | EA triages and routes manually | hiring, benefits, employee, PTO, onboarding new hire |
| Client Success | Exec review required | onboarding, renewal, client request, account, client issue |
| Scheduling | EA handles directly | calendar, meeting, reschedule, availability |
| Operations | EA handles directly | general, admin, logistics |
| Vendors | EA handles directly | vendor, contract follow-up, coordination |
| Legal | Exec review required | legal, contract, notice, counsel |
| Everything else | EA triages, escalates if unsure | - |
